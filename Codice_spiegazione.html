Immagini vettoriale

Le immagini raster sono composte da un numero specifico di pixel, ognuno di un colore preciso che insieme formano l'mmagine. Sono buone per fotografie e immagini complesse, se ingrandite diventano sgranate Non si possono manipolare con JS, problemi con CSS I formati tipici sono JPG, JPEG, PNG, GIF ecc

Le immagini vettoriali non usano pixel, ma forme matematiche: linee, curve, poligoni. Si possono ingrandire all'infinito e non saranno sgranate perchè verranno inseriti i pixel necessari, sono file più leggeri. Ottime per loghi, icone, illustrazioni e scritte. I formati tipici sono SVG, AIM, EPS, PDF.

UN'immagine SVG è scritta con il codice XML Il codice XML (eXtensible Markup Language) è un linguaggio usato per strutturare, organizzare e trasportare dati in modo chiaro e leggibile sia da persone che da computer. Serve a scambiare dati tra programmi sito server ecc (approfondire)

Immagini responsive

Le immagini responsive sono immagini che si adattano automaticamente alle dimensioni dello schermo o del dispositivo in cui vengono visualizzate.

Occorre approfondire ma essenzialmente o si fanno con il CSS oppure su html si caricano diverse versioni della stessa immagine e il browser caricherà quella più adatta per il dispositivo utilizzato

Span

Il tag span è un elemento inline utile per raggruppare un elemento di testo all'interno di un paragrafo o di un altro contenitore senza modificarne visivamente l'aspetto
Tuttavia è utile per cambiare colore o dimensione o altri elementi.

Tag semantici

I tag semantici sono elementi HTML che descrivono chiaramente il significato del loro contenuto per i browser, i lettori e i motori di ricerca.
Utile per migliorare l'accessibilità, l'indicizzazione nei motori di ricerca e la leggibilità

Il tag semnantico <header> si usa per l'intestazione

Il tag semnantico <footer> si usa per la parte finale della pagina come contatti, policy ecc

Il tag semnantico <main> si usa per il contenuto principale della pagina

Il tag semnantico <section> si usa per una determinata sezione tematica

Il tag semnantico <article>per un contenuto extra o laterale

Il tag semnantico <aside>per un contenuto autonomo

Il tag semnantico <nav>per permettere al lettore di navigare agilmente attraverso le sezioni del sito.

Il tag semnantico <time>si usa per indicare una data o un'ora in modo leggibile per umani e motori di ricerca

Struttura della tabella

La struttura di una tabella serve a organizzare i dati in modo logico.

thead contiene le intestazioni.

tbody contiene i dati principali.

tfoot contiene eventuali totali o riepiloghi.

Stile della tabella

Lo stile si puà modificare con colgroup (ma soltanto background color,width, visibility)

Il border si può modificare agendo direttamente nel th/td

Tips: sei si vuole spazio tra testo e bordo cella → padding su td

Se si vuole spazio tra tabella e resto della pagina → margin su table

Se si vuole spazio tra bordo esterno e celle → padding su table + bordo visibile

Form, input, checkbox e bottoni

I form sono uno strumento di raccolta dati (moduli di contatto, newsletter, moduli da compilare) in cui l'utente può inserire dati che poi verranno inviati attraverso l'attributo action ad un server utilizzando uno specifico metodo HTTP che viene scelto attraverso l'attributo method (dialog, get o post)

Con il metodo get i valori digitati dall'utente sono visibili nella stringa dell'URL quando viene inviato.

La lunghezza dell'URL è limitata a 2048 caratteri.

Con il metodo post i valori vengono inviati nel corpo della richiesta, di conseguenza non sono visibili nell'URL ed è quindi indicato per il trattamento di dati sensibili

L'attributo post non ha limiti di dimensione e può essere utilizzato per inviare grandi quantità di dati

Attributi

  • Checked: risulta già checkata di default
  • Disabled: disabilita il campo, non viene inviato nel form
  • Readonly: campo di sola lettura, l'utente non può modificarlo
  • Required: campo obbligatorio, impedisce l'invio del form se il campo non è compilato
  • Value: valore effettivo del campo che viene inviato al server
  • Value: valore preimpostato del campo, può essere modificato dall'utente
  • Multiple: fa selezionare più elementi
  • Placeholder: suggerisce all'utente cosa scrivere in quel determinato campo, scompare quando l'utente scrive
  • Maxlenght: indica la lunghezza massima di caratteri che si possono inserire
